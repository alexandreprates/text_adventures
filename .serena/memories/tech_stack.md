# Tech Stack

- Language: Ruby.
- Docker target: `ruby:2-alpine`; no `.ruby-version` is checked in.
- Package manager/tooling: Bundler expected by Dockerfile and project commands; local app config installs gems into `vendor/bundle`.
- Runtime dependencies from `Gemfile`: `rake`.
- Test dependencies from `Gemfile` test group: `rspec`.
- `spec/spec_helper.rb` loads Coveralls only when the gem is available; coverage upload tooling is optional and not part of the required bundle.
- No RuboCop, formatter config, gemspec, application framework, or required Guard watcher dependency is currently present.
- `Gemfile.lock` is generated by dependency installation and should be kept if the project wants reproducible local dependency versions.
