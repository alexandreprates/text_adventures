# Content Core

- Durable content files live under `data/`:
  - `items.yml`: weapons, armor, potions, tomes, and item stats/effects.
  - `shops.yml`: shop inventories and grouped stock.
  - `creatures.yml`: enemies, attacks, loot, status effects, and XP rewards.
  - `dungeon_blocks.yml`: block tile layouts and dungeon generation content.
- `ContentCatalog` loads YAML and builds domain objects; prefer editing YAML for content-driven gameplay changes.
- Dungeon YAML tiles should represent terrain/layout only. Runtime render markers are generated by dungeon state, not authored into block tiles.
- Current runtime dungeon markers are `x`, `E`, `@`, `P`, `<`, `>`, `.`, `#`, and `?`.
- After content edits, run matching specs under `spec/domain` and/or `spec/data`, then the full suite.
- Enemy visual assets and manifest live under `frontend/public/assets/enemies/`; keep creature IDs/content aligned with frontend asset expectations when changing enemies.
- Use deterministic seeds for validating changes to dungeon generation, loot, encounters, and combat balance.
